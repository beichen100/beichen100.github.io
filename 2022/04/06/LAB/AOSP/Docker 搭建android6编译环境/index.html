Docker 搭建android编译环境

2022-04-06 AOSP Docker, android Comments Word Count: 355(words) Read Count: 1(minutes)

由于服务器环境不适用于android6的源码编译，所以考虑用Docker搭建AOSP编译环境。

docker搭建android6有现成docker镜像，可参考：
http://qiushao.net/2019/11/14/Linux/docker-aosp-build-env/

  1. 安装docker

  2. 编译docker镜像

    按上面的步骤操作完之后，应该生成一个 aosp_builder:V1.0 的docker 镜像了,构建出来的镜像用户名为 builder。

  3. 创建容器：docker run -it --name aosp_builder -v ~/android6:/home/builder/android6 -u builder aosp_builder:V1.0 /bin/bash

    其中,-v ~/source:/home/builder/android6 参数是将宿主机目录~/android6映射到 docker 容器的/home/builder/android6目录。因为我宿主机上 Android 的源码放在 ~/android6这个目录里面了。
    -u builder表示使用 builder 用户登录，不指定的话，会使用 root 用户登录，最好不要用 root 登录，用 root 用户登录的话，编译时产生的文件用户属主就是 root 了，在宿主机上要编辑或删除这些编译生成的文件就得用 sudo 了。

  4. 进入docker镜像环境

    然后我们就可以按照 Android 的标准编译流程开编了
