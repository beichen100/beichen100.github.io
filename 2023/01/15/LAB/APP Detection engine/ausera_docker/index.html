ausera_docker 部署

2023-01-15 APP Detection engine android，Docker Comments Word Count: 560(words) Read Count: 2(minutes)

  1. docker pull james0818/jdk1.8.0_241:latest , 发现 docker 没启动，直接 sudo service docker start

  2. 发现 docker 命令老需要加 sudo ，所以添加自己用户到 docker 组

  3. 查看 pull 下来的本地镜像 docker images， 然后重命名镜像，docker tag oldname:tag newname:tag ，删除镜像， docker rmi image_name: tag

  4. 创建容器 docker run –name 容器名 镜像名 , 删除容器 docker rm 容器名, 查看运行的容器 docker ps , 查看所有容器 docker ps -a

  5. 以交互模式创建镜像 docker run -it 镜像名 /bin/bash, 以交互模式进入容器 docker start -i 容器名

  6. 容器文件传输 docker cp 本地文件 容器名: 路径

  7. 打包容器成镜像 docker commit 镜像名

  8. 打包镜像成本地文件 docker save 镜像名>文件名.tar

  9. 搭建本地私有docker 仓库

  10. 给镜像打标签后上传到私有仓库

    • 将镜像改名为 IP/镜像名, docker tag nginx:latest 192.168.50.128:5000/nginx
    • 上传 docker push 192.168.50.128:5000/nginx
    • 获取私有仓库列表查看是否上传成功, curl -XGET http://192.168.50.128:5000/v2/_catalog, 会返回镜像名
    • 从私有仓库下载镜像 docker pull 192.168.50.128:5000/nginx

  11. 其他机器pull 镜像时，需要先 vim etc/docker/daemon.json，添加

python2.7 apk-engine.py /root/ausera/ /usr/local/jdk/jdk1.8.0_241 /root/ausera/engine-configuration/libs/android-platforms
